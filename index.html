Semantic HTML Reference Sheet

Semantic HTML is used by developers to provide a greater meaning to the code we write. We can use elements to both remind us, and to tell browsers, exactly what the content is or does.

Semantic HTML uses elements that describe its placement or use by its own name. Some of these elements are <code>, <nav> or <main>, which are far more descriptive than the old <div> and <span> tags, though these older elements still have valid use cases.

Semantic HTML also uses elements that provide far greater context on content. These would be elements such as <audio>, <video> or <figure>.

Tag Description Additional Uses
<header> Describes content at the top of the page, within the <body> element. May include <nav> or <img> elements.
<nav> Used to encapsulate navigational links. Frequently placed in header or footer areas.
<main> For the main content between the header/navigation and footer areas Encapsulates the primary content.
<footer> For the footer content at the base of the page. Can include navigational, copyright or contact information.
<video> For implementing video to the site 'src-""' and Requires a closing tag
<audio> Allows us to implement audio into the website 'src-""' Requires a closing tag.
<embed> Can be used for any (other) type of media This tag is universal and is self-closing.
<figure> Used to encapsulate various media like images/diagrams/code snippets Ideally used in conjuction with <figcaption> (see below)
<figcaption> To describe the element within the <figure> (see above) Use within the figure elements.
<section> Define relevent sections with a document Headings, chapters and themes can all be defined with this element.
<article> Use around very specific themes. For quotes, code snippets, or other content that makes sense on its own.
<aside> Can be used to earmark additional information that might enhance (but is not critical to) another element. Use in a sidebar, or in a location not obstructing the main content.
